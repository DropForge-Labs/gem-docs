Introduction

This document pertains to a baseline audit performed on the https://www.eastman.org (Eastman Museum) web property. This audit was performed in order to provide insight into the pain-points, inefficiencies, and successes of the existing Eastman.org Drupal implementation. 

The audit process delivers tremendous value including, but not limited to, the following:

  • Identifies interim risks that can be mitigated if deemed necessary (compliance issues, security issues, brand integrity risks)
  • Provides architectural insight necessary to develop a content migration plan
  • Familiarizes DropForge team with existing content, workflows and editorial processes
  • Identifies previous failures to prevent future inefficiencies and pain-points with the new site

Analysis of the existing Drupal 7 website is presented in the table below.  The format provided includes a table of findings, a list of action items, and closing remarks. The facts and findings were discovered during the investigative portion of the audit process, including the code review, current site configuration, and basic hosting infrastructure.  Finally, recommendations will be presented as suggested resolutions to mitigate and/or better understand the identified weaknesses.

Any questions regarding the material provided will be answered by the C/O residing with DropForge Labs.  Please also be sure to review the referenced appendixes (if applicable) relating to their relevant sections for information regarding how some of the facts and findings were obtained.

Action Plan

Given the fact that this site will be retired and archived relatively soon, it is in both parties best interest to minimize the allocation of resources towards improving the Drupal 7 site.  However, we have identified the following list of items that may be worth investing in the improvement of the existing Drupal 7 site.

  1. Perform security updates
  2. Enable css / js caching
    1. Only if it doesn’t break existing site
  3. Fix footer contact link
  4. Limit database logging to 1K rows
  5. Disable unused modules
    1. Only if it doesn’t break existing site
  6. Evaluate / Remove non-Drupal items in docroot
    1. To be performed by GEM Team
  7. Remove Drupal core TXT files from docroot
    1. To be performed by GEM team

Closing Remarks

From the public facing user’s perspective, the site is well-built and organized efficiently. The site is somewhat responsive and fast enough that it’s not a burden on mobile devices. Based on our Code Review, the code-base seriously lacks maintainability.  This is primarily due to previous development teams failing to use industry and Drupal best practices.  Replacement of the legacy Drupal 7 Site with a highly optimized Drupal 8 build will result in a codebase that's maintainable, more performant, and experience a reduced total-cost-of-ownership.

Migration of existing content will be somewhat demanding due to the varying architecture of Drupal features, however still very capable.
